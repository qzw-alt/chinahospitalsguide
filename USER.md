## 工作偏好

### 任务跟进方式
- **重要**: 没有明确回复 ≠ 任务取消
- **规则**: 未完成的重要任务需要持续跟进提醒
- **原因**: 用户可能忙、睡着或忘记回复，但不代表任务不需要做

### 任务状态分类（必须记录）
| 状态 | 含义 | 我的行动 |
|------|------|---------|
| **待处理** | 用户还没开始处理 | 每天提醒 |
| **进行中** | 用户正在处理 | 每2-3天跟进进度 |
| **推迟到明天** | 今天处理不了，明天处理 | 记录到文件，第二天提醒 |
| **部分完成** | 处理了部分，还有剩余 | 记录已完成和未完成，跟进剩余 |
| **处理不了** | 确定无法处理 | 询问原因，记录并归档 |
| **已完成** | 明确说完成了 | 从HOT删除，记录到日志 |

### 部分完成的任务处理
**场景**: 给了4个建议，用户只处理了2个，还有2个没处理
**我的行动**:
1. 记录到HOT: "已完成: 建议1、2 | 待处理: 建议3、4"
2. 1-2天后提醒: "之前讨论了4个点，你处理了1和2，还有3和4需要处理吗？"
3. 直到用户明确说"处理完了"或"不处理了"

### 推迟任务的处理
**场景**: 用户说"今天处理不了，明天处理"
**我的行动**:
1. 记录到HOT，标注: "推迟到2026-03-17"
2. 第二天主动提醒: "昨天你说今天处理XX任务，现在方便处理吗？"
3. 如果又说推迟，更新日期，继续跟进

### 记忆锚点
> "用户说'明天处理' → 记录日期 → 第二天提醒"
> "部分完成 → 记录剩余项 → 跟进剩余"
> "没有明确说'完成'或'取消' → 任务还在"
