## 🚀 网站部署流程（标准SOP）

### 每次修改网站后必须执行：

```bash
# 1. 编辑文件（在 /root/.openclaw/workspace/ 目录）

# 2. 推送到网站仓库（只传网站文件）
cd /tmp
git clone https://github.com/qzw-alt/chinahospitalsguide.git website-temp
cp -r /root/.openclaw/workspace/blog/* website-temp/blog/
cp /root/.openclaw/workspace/index.html website-temp/
cp /root/.openclaw/workspace/*.html website-temp/ 2>/dev/null || true

# 3. 提交并推送
cd website-temp
git add .
git config user.email "demi@openclaw.ai"
git config user.name "Demi"
git commit -m "更新描述"
git remote set-url origin https://TOKEN@github.com/qzw-alt/chinahospitalsguide.git
git push origin master

# 4. 备份到 demi 仓库（所有文件）
# 5. 等待2-5分钟部署
# 6. 验证网站更新
```

## ⚠️ 常见错误（避免重复）

### 错误1：推送到错误仓库
❌ 推送到 `demi` 仓库（这是备份仓库！）  
✅ 推送到 `qzw-alt/chinahospitalsguide` 仓库（网站仓库）

### 错误2：只在 docs/ 修改，不同步到根目录
❌ 只修改 `docs/blog/` 下的文件  
✅ 修改后执行 `cp -r docs/* .` 同步到根目录

### 错误3：推送后立即检查网站
❌ 推送完立刻刷新网站  
✅ 等待2-5分钟后再检查

## 📝 博客文章风格更新 (2026-03-21)

### 旧风格（已过度使用）
- ❌ 主打"省钱"角度
- ❌ "Save 70%", "$3000 vs $20000" 反复出现
- ❌ 让人觉得"便宜=不安全"

### 新风格（现在起执行）
- ✅ 主打"安全+专业"
- ✅ 强调JCI认证、国际培训、循证数据、真实成功率
- ✅ 省钱作为自然附带结果，一句话带过
- ✅ 引用具体医生资质、医院排名、研究数据
- ✅ 患者案例侧重"重获健康"而不是"省了多少钱"

### 文章结构优先级
1. 核心卖点：安全性 > 专业性 > 成本合理性
2. 数据支撑：成功率、认证、医生背景
3. 省钱只是附加优势，提一句就好
