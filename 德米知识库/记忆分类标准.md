# 🧠 记忆分类标准（德米使用指南）

> **创建日期**: 2026-03-16  
> **用途**: 指导德米正确分类和存储记忆

---

## 📊 三层记忆架构

### 1. 🔥 HOT (热记忆) - `memory/hot/HOT_MEMORY.md`
**用途**: 当前活跃任务和临时信息  
**生命周期**: 任务完成后删除或归档  
**更新频率**: 每次会话

**存放内容**:
- 当前进行中的任务
- 今天的待办清单
- 临时上下文信息
- 会话中的临时决策

**示例**:
```
- [ ] 写3篇博客文章
- [ ] 提交sitemap到Google
- 当前在等用户确认XX
```

---

### 2. 🌡️ WARM (温记忆) - `memory/warm/*.md`
**用途**: 稳定配置、用户偏好、错误教训  
**生命周期**: 长期保留，信息变更时更新  
**更新频率**: 信息变更时

**存放内容**:
- 用户基本信息（名字、时区、偏好）
- 项目历史决策和背景
- **错误教训和踩坑记录** ⚠️
- 稳定的工作流程（非SOP）
- 常用的配置和设置

**示例**:
```
- 伟烨的时区: Asia/Shanghai
- 医疗旅游项目决策: 专注B2C而非B2B
- 定价策略: Starter $30 / Standard $299 / Premium $799
```

**文件命名**:
- `WARM_MEMORY.md` - 通用偏好
- `医疗旅游_项目背景.md` - 项目决策
- `龙康劲_运营策略.md` - 其他项目

---

### 3. ❄️ COLD (冷记忆) - `MEMORY.md` + `memory/cold/*.md`
**用途**: 长期归档，核心知识  
**生命周期**: 永久保留  
**更新频率**: 重要信息变更时

**存放内容**:
- 技能SOP和操作流程
- 核心项目信息（仓库地址、域名等）
- 重要的经验教训
- 踩坑记录和解决方案

**示例**:
```
## 博客写作SOP
1. multi-search-engine 搜索
2. tavily-search 深度研究
3. content-research-writer 写作
4. humanizer 去AI化

## 医疗旅游核心信息
- 网站仓库: qzw-alt/demi
- 网站域名: chinahospitalsguide.com
- 部署流程: ...
```

---

## 📝 每日日志

**位置**: `memory/YYYY-MM-DD.md`  
**用途**: 当天工作记录  
**生命周期**: 15天后归档到 COLD

**存放内容**:
- 今天做了什么
- 今天的决策和原因
- 遇到的问题和解决方案
- 明天的计划

---

## 🎯 快速判断指南

| 问题 | 答案 | 放哪里 |
|------|------|--------|
| 这是操作流程/技能SOP吗？ | 是 | **COLD (MEMORY.md)** |
| 这是核心项目信息吗？（仓库、域名） | 是 | **COLD (MEMORY.md)** |
| 这是历史决策/背景吗？ | 是 | **WARM** |
| 这是用户偏好/配置吗？ | 是 | **WARM** |
| 这是错误教训/踩坑记录吗？ | 是 | **WARM** ⚠️ |
| 这是当前任务/待办吗？ | 是 | **HOT** |
| 这是临时信息吗？ | 是 | **HOT** |
| 这是当天工作记录吗？ | 是 | **每日日志** |

---

## ❌ 常见错误

### 错误1: 把SOP放在WARM
❌ `memory/warm/博客写作SOP.md`  
✅ 应该放在 `MEMORY.md` 或 `德米知识库/`

### 错误2: 把临时任务放在WARM
❌ 在WARM里记录"今天待办"  
✅ 应该放在 `HOT_MEMORY.md`，做完删除

### 错误3: 把核心信息放在HOT
❌ 在HOT里记录"网站仓库地址"  
✅ 应该放在 `MEMORY.md` 长期记忆

## 🔔 任务跟进规则（重要！）

### 未完成任务的跟进

**原则**: 用户没有明确回复 ≠ 任务取消

**跟进规则**:
1. **重要任务**（如注册账号、申请接口）→ 每天提醒一次
2. **普通任务**（如写文章）→ 每2-3天提醒一次
3. **直到**用户明确说：
   - "完成了" → 从 HOT 删除
   - "取消了" → 从 HOT 删除并记录原因
   - "推迟到XX" → 更新截止日期

**常见情况**:
- 用户没回复 → 继续提醒
- 用户说"等会儿" → 1小时后再次提醒
- 用户说"明天" → 明天同一时间提醒
- 用户睡着了/忙 → 下次会话时提醒

**记忆锚点**:
> "没有明确回复 = 任务还在，继续跟进"
